Introduction à XHTML

Bienvenue ! Ce guide vous présente les concepts fondamentaux de XHTML et vous prépare à créer des documents web conformes aux standards.

Qu'est-ce que XHTML ?

XHTML (eXtensible HyperText Markup Language) est une reformulation de HTML en tant qu'application XML. Il combine la flexibilité de HTML avec la rigueur de XML, créant ainsi un langage de balisage plus strict et plus cohérent.

Définition Officielle

Selon le W3C, XHTML est "une famille de types de documents et de modules actuels et futurs qui reproduisent, sous-ensemblent et étendent HTML, reformulé en XML 1.0".

Historique et Évolution

HTML (1991-1999)

  • HTML 1.0 (1991) : Version initiale
  • HTML 2.0 (1995) : Première standardisation
  • HTML 3.2 (1997) : Ajout de tableaux et applets
  • HTML 4.01 (1999) : Séparation contenu/présentation

XHTML (2000-présent)

  • XHTML 1.0 (2000) : HTML 4 en XML
  • XHTML 1.1 (2001) : Version modulaire stricte
  • XHTML 2.0 (abandonné) : Trop radical
  • HTML5 (2014) : Retour à HTML avec XHTML optionnel

HTML vs XHTML : Les Différences

Aspect HTML XHTML
Syntaxe Permissive, tolère les erreurs Stricte, aucune erreur tolérée
Balises Majuscules ou minuscules Uniquement minuscules
Fermeture Optionnelle pour certaines balises Obligatoire pour toutes
Attributs Peuvent être sans valeur Doivent toujours avoir une valeur
Guillemets Optionnels pour les valeurs Obligatoires
Validation Recommandée Obligatoire
Namespace Non requis Requis (xmlns)
DOCTYPE Recommandé Obligatoire

Les Règles Strictes de XHTML

XHTML impose des règles strictes qui garantissent la cohérence et la validité du code :

  • Toutes les balises en minuscules : <div> et non <DIV>
  • Fermeture obligatoire : <p>texte</p> et non <p>texte
  • Balises auto-fermantes : <br /> et non <br>
  • Attributs entre guillemets : class="menu" et non class=menu
  • Attributs avec valeurs : checked="checked" et non checked
  • Hiérarchie respectée : Pas de <p> dans un autre <p>
  • DOCTYPE obligatoire : Déclaration en début de document
  • Encodage déclaré : Spécification de l'encodage des caractères
  • Namespace XHTML : xmlns="http://www.w3.org/1999/xhtml"
  • Document bien formé : Structure XML valide

Exemples de Code

❌ HTML Permissif (Non valide en XHTML)

<!-- Balises en majuscules -->
<DIV>
    <!-- Balise non fermée -->
    <P>Ceci est un paragraphe
    
    <!-- Attribut sans guillemets -->
    <IMG src=image.jpg>
    
    <!-- Attribut booléen sans valeur -->
    <INPUT type=checkbox checked>
</DIV>

✅ XHTML Strict (Valide)

<!-- Balises en minuscules -->
<div>
    <!-- Balise correctement fermée -->
    <p>Ceci est un paragraphe</p>
    
    <!-- Attribut avec guillemets -->
    <img src="image.jpg" alt="Description" />
    
    <!-- Attribut booléen avec valeur -->
    <input type="checkbox" checked="checked" />
</div>

Pourquoi Utiliser XHTML ?

Avantages

  • Code plus propre : Syntaxe stricte = code cohérent
  • Meilleure compatibilité : Fonctionne sur tous les navigateurs
  • Facilite le traitement : Peut être parsé comme XML
  • Détection d'erreurs : Les erreurs sont immédiatement visibles
  • Accessibilité : Structure claire pour les lecteurs d'écran
  • Maintenance : Code plus facile à maintenir

Inconvénients

  • Plus strict : Moins tolérant aux erreurs
  • Verbeux : Code plus long (ex: <br />)
  • Courbe d'apprentissage : Nécessite de la rigueur
  • Type MIME : Peut causer des problèmes de serveur
  • Moins populaire : HTML5 est devenu le standard

Quand Utiliser XHTML ?

Cas d'Usage Recommandés

  • Applications d'entreprise : Nécessitant une validation stricte
  • Traitement XML : Quand le document doit être parsé comme XML
  • Intégration avec XML : Mélange de XHTML avec d'autres XML (SVG, MathML)
  • Formation : Apprendre les bonnes pratiques de codage
  • Accessibilité : Garantir une structure sémantique correcte

Configuration Requise

DOCTYPE pour XHTML 1.1

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" 
    "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">

Balise HTML avec Namespace

<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="fr">

En-tête Meta Complète

<head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    <title>Titre de la Page</title>
</head>

Structure Minimale d'un Document XHTML

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" 
    "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="fr">
<head>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    <title>Ma Première Page XHTML</title>
</head>
<body>
    <h1>Bienvenue en XHTML</h1>
    <p>Ceci est un document XHTML valide.</p>
</body>
</html>

Points Clés à Retenir

  • XHTML est une version stricte de HTML basée sur XML
  • Toutes les règles doivent être respectées (pas de tolérance aux erreurs)
  • Le code XHTML est plus propre et plus maintenable
  • La validation est obligatoire, pas optionnelle
  • XHTML prépare à comprendre XML et les standards web
  • Bien que HTML5 soit devenu le standard, XHTML reste pertinent pour l'apprentissage
