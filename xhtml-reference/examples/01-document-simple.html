Document XHTML Simple - Exemple Complet

Cet exemple démontre la structure de base d'un document XHTML 1.1 valide. Chaque élément respecte les règles strictes de XHTML.

1. Déclaration DOCTYPE

Le document commence par la déclaration DOCTYPE qui indique au navigateur que nous utilisons XHTML 1.1 :

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">
Note : Cette déclaration est obligatoire en XHTML 1.1 et doit être la première ligne du document.

2. Élément Racine <html>

L'élément <html> contient deux attributs obligatoires :

  • xmlns : Définit l'espace de nommage XHTML
  • xml:lang : Spécifie la langue du document
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="fr">

3. Section <head>

La section <head> contient les métadonnées du document :

<head> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /> <title>Titre de la Page</title> </head>
Important : Notez que la balise <meta /> est auto-fermante avec /> à la fin. C'est une règle XHTML stricte.

4. Section <body>

Le <body> contient tout le contenu visible de la page :

<body> <h1>Mon Premier Document XHTML</h1> <p>Ceci est un paragraphe.</p> </body>

5. Exemple de Rendu

Voici comment le contenu s'affiche dans le navigateur :

Mon Premier Document XHTML

Ceci est un paragraphe.

6. Document Complet

Voici le code source complet de ce document :

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd"> <html xmlns="http://www.w3.org/1999/xhtml" xml:lang="fr"> <head> <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" /> <title>Mon Premier Document XHTML</title> </head> <body> <h1>Mon Premier Document XHTML</h1> <p>Ceci est un paragraphe.</p> </body> </html>

7. Points Clés à Retenir

  • ✓ DOCTYPE obligatoire en première ligne
  • ✓ Namespace xmlns dans la balise <html>
  • ✓ Toutes les balises en minuscules
  • ✓ Toutes les balises fermées (même les auto-fermantes avec />)
  • ✓ Tous les attributs entre guillemets
  • ✓ Structure hiérarchique respectée

8. Validation

Pour vérifier que votre document XHTML est valide, utilisez le validateur W3C.

Exercice : Essayez de créer votre propre document XHTML en suivant cette structure. Validez-le ensuite avec le validateur W3C.
